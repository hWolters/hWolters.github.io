<div class="legacy-content"><p>In this blogpost I will share some tips for working with Jupyter Notebooks. Those tips greatly improved my productivity when working with Jupyter Notebooks and I wish someone would have told me earlier. The two main topics of this post are extensions and magic commands.</p>&#10;<h2 id="jupyter-extensions">Jupyter Extensions</h2>&#10;<p>Have you ever missed a feature in your Jupyter Notebook that IDEs have? E.g. you were hoping for autocompletion or automatically formatting code? Then there might be a Jupyter Notebook extension for you. Jupyter extensions simply add features to your Jupyter Notebook, similar to what browser plug-ins do to browsers. Just pip install <a href="https://jupyter-contrib-nbextensions.readthedocs.io/en/latest/install.html"><code>jupyter-contrib-nbextensions</code></a>. On your <code>localhost:8888</code> you will now find a tab called Nbextensions. There, you can select the extensions you would like to use by checking the boxes. Usually, you have to restart your kernel for the extension to work. In the following, I’ll show you my five favorite extensions.</p>&#10;<p><img loading="lazy" decoding="async" src="/post/2020-07-jupyter-notbooks-boost-your-productivity/JupyterExtensions.png" alt="Illustration for Jupyter Notebooks: Boost your productivity with Extensions and Magic Commands">&#10;<br></p>&#10;<h3 id="notify">Notify</h3>&#10;<p>Have you ever waited for some time consuming code to execute, switched tabs and then forgot about it? Or you kept staring at your notebook to wait for the execution of the cell to finally have finished? No need for that with the <code>Notify</code> extension: It displays a web notification, when a cell has run.&#10;After you added this extension, you will see a new dropdown menu in your notebook’s toolbar. You can choose, if and when you want to be notified. Your choices are: Disabled, 0, 5, 10 and 30. If you select for example 5, you will get a notification, if the kernel is busy longer than 5 seconds. I am using the extension with Safari and it works great. It should also work with Firefox and Chrome browsers.</p>&#10;<p><img loading="lazy" decoding="async" src="/post/2020-07-jupyter-notbooks-boost-your-productivity/JupyterToolBarNotify.png" alt="Illustration for Jupyter Notebooks: Boost your productivity with Extensions and Magic Commands">&#10;<br></p>&#10;<h3 id="snippets">Snippets</h3>&#10;<p>Do you sometimes find yourself typing the same code snippets over and over again? Maybe you copy paste your database connection strings, some of your most used functions or you always load the same packages? Then the <code>Snippets</code> extension may be useful for you.</p>&#10;<p>You have to add your snippet as a new JSON block to the list of existing snippets in <code>snippets.json</code>. You can find this file in your jupyter directory with <code>jupyter --data-dir</code> and then going to <code>/nbextensions/snippets/snippets.json.</code> There are already 2 example snippets. You can add your snippets there by just appending to the <code>.json</code> file in the same format. For example, to add a new snippet that imports your favorite data science packages, just add it to the json file.</p>&#10;<pre><code>        {&#10;            "name" : "ds imports",&#10;            "code" : [&#10;                "import pandas as pd",&#10;                "import numpy as np",&#10;                "import matplotlib.pyplot as plt",&#10;                "import seaborn as sns"&#10;            ]&#10;        }&#10;</code></pre><p>After you added a snippet, it shows up in the tool bar in the drop down menu snippets. I use this a lot for database configurations and loading the standard packages.</p>&#10;<p><img loading="lazy" decoding="async" src="/post/2020-07-jupyter-notbooks-boost-your-productivity/JupyterToolBarSnippet.png" alt="Illustration for Jupyter Notebooks: Boost your productivity with Extensions and Magic Commands">&#10;<br></p>&#10;<h3 id="autopep8">Autopep8</h3>&#10;<p>If you neglected coding style guidelines Autopep8 could help you. It automatically reformates your code to meet the <a href="Phttps://www.python.org/dev/peps/pep-0008/">PEP 8 standards</a>, Python’s most popular style guide. If you do not already have it, you need to <code>%%pip install autopep8</code>.&#10;Then you can autoformat your code by pressing <code>Alt-A</code> for a cell or <code>Alt-Shift-A</code> for the entire notebook. Or you could also select a cell and click on the little hammer icon on the right of your toolbar and your code will be prettified!</p>&#10;<p><img loading="lazy" decoding="async" src="/post/2020-07-jupyter-notbooks-boost-your-productivity/JupyterToolBarFormat.png" alt="Illustration for Jupyter Notebooks: Boost your productivity with Extensions and Magic Commands">&#10;<br></p>&#10;<h3 id="hinterland">hinterland</h3>&#10;<p>Autocompletion in Jupyter Notebooks exists, but it is cumbersome: You need to press <code>tab</code> everytime you need it. So I really missed the quick autocompletion with every keypress  until I found out about <code>hinterland</code>. With <code>hinterland</code> you will have the autocompletion with every key press. You just need to add the extension.</p>&#10;<h3 id="executetime">ExecuteTime</h3>&#10;<p>Sometimes it is interesting to know how long the execution of entire cells took. Of course you could use <code>%%timit</code> in every cell. Or you save some time by using the <code>ExecuteTime</code> extension. For each cell, you will automatically see when the execution started and how long it took.</p>&#10;<p>The following table summarizes my top 5 extensions. Check out the <a href="https://jupyter-contrib-nbextensions.readthedocs.io/en/latest/nbextensions.html">official documentation for all nbextensions</a> for more.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>Extension</th>&#10;<th>what it does</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Autopep8</td>&#10;<td>autoformat code</td>&#10;</tr>&#10;<tr>&#10;<td>hinterland</td>&#10;<td>autocompletion</td>&#10;</tr>&#10;<tr>&#10;<td>ExecuteTime</td>&#10;<td>show execution time of each cell</td>&#10;</tr>&#10;<tr>&#10;<td>Notify</td>&#10;<td>get browser notification when cell has run</td>&#10;</tr>&#10;<tr>&#10;<td>Snippets</td>&#10;<td>insert code snippets from drop down menu</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<h2 id="magic-commands">Magic Commands</h2>&#10;<p>Magic commands are shortcuts that significantly extend a notebook’s capabilities. The Python kernel uses the <code>%</code> syntax element for magic commands. You can use them line and cellwise. In line-wise, all given commands must start with the <code>%</code> character. For an entire cell, all commands start with<code>%%</code>.</p>&#10;<p>Let’s look at my personal top 5 magic commands:</p>&#10;<h3 id="history"><code>%history</code></h3>&#10;<p>So you did something super smart but then you deleted your cell and you forgot to hit the undo button and you can’t remember how you did it? <code>%history</code> shows you the history of all commands of your current notebook since the kernel started.</p>&#10;<h3 id="matplotlib-inline"><code>%matplotlib inline</code></h3>&#10;<p>During your data analysis you may create a lot of plots. If you insert <code>%matplotlib inline</code> in a cell at the beginning of your notebook, your plots will be directly shown below the code cell that produced it.</p>&#10;<h3 id="system"><code>%%system</code></h3>&#10;<p>You briefly want to check your directory and directory contents without using your terminal separately? You can use shell commands from your Jupyter Notebook with <code>%system</code> for a single line of shell script or <code>%%system</code> at the beginning of your cell for using the shell in the entire cell. <code>%%system</code> let’s you use your Jupytper notebook cell like your terminal.</p>&#10;<h3 id="latex"><code>%%latex</code></h3>&#10;<p>This one comes in handy if you want to include mathematical formulas in your notebook. Formulas in Markdown Syntax are limited and cumbersome. Just insert a latex cell and type your formulas with latex syntax.</p>&#10;<h3 id="capture-"><code>%%capture </code></h3>&#10;<p>Let’s say you have some time consuming code running in a cell. Closing the Jupyter window won’t prevent the cell from running. However, you would lose your printed output, which may be of interest for you. If you start your cell with <code>%%capture &lt;someName&gt;</code>, the cell output will be saved and you can retrieve it later, even if you closed your notebook.</p>&#10;<pre><code>%%capture output&#10;# Time-consuming code here&#10;# show all interim print results&#10;output.show()	&#10;</code></pre><p>Let’s summarize my favorite magic commands. If you want to find out more about magic commands check out the <a href="https://ipython.readthedocs.io/en/stable/interactive/magics.html">IPython Documentation</a></p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>Macic command</th>&#10;<th>what it does</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td><code>%history</code></td>&#10;<td>see all commands in this notebook</td>&#10;</tr>&#10;<tr>&#10;<td><code>%matplotlib inline</code></td>&#10;<td>shows plots inline</td>&#10;</tr>&#10;<tr>&#10;<td><code>%%system</code></td>&#10;<td>use the shell directly from your nb</td>&#10;</tr>&#10;<tr>&#10;<td><code>%%latex </code></td>&#10;<td>insert latex syntax</td>&#10;</tr>&#10;<tr>&#10;<td><code>%%capture </code></td>&#10;<td>save printed output of a cell</td>&#10;</tr>&#10;</tbody>&#10;</table></div>
