<div class="legacy-content"><p>I write this blogpost for someone, who has never worked with R Markdown. After you read this post, you will</p>&#10;<ol style="list-style-type: decimal">&#10;<li><p>understand why R Markdown may be useful for your daily work as a student, researcher, analyst or data scientist.</p></li>&#10;<li><p>understand the basic structure of an R Markdown document and how you can get startet.</p></li>&#10;</ol>&#10;<p>I strongly encourage everybody working with R to use R Markdown. I promise, it will make your life so much easier. Here are some examples when R Markdown can be very useful:</p>&#10;<ul>&#10;<li><p><strong>Results sharing:</strong> You just did a great analysis and want to share your results with your team. Then you start copy pasting your Output into Excel, Word or PowerPoint. Don’t do that! It takes so much time and you might forget to copy paste the most curret version of a plot or an analysis. R Markdown can automatically convert your results to many formats such as html, word and pdf. No more copy pasting necessary! This will not only save you lot’s of time, it is also less error prone: With R Markdown, everything is up to date automatically!</p></li>&#10;<li><p><strong>Data exploration</strong> You are looking at various summary statistics and plots to understand your data. Without R Markdown, you might scroll through your console and use the back and forward button in the R Studio Viewer. This is cumbersome! You can use R Markdown like a Notebook and see multiple output tables and plots without having to use the RStudio Viewer back buttons. At the same time, you can see the code, that produced the plot. Which brings me to my next point:</p></li>&#10;<li><p><strong>Replicable Analyses:</strong> In general, all your results should be repoducible. If you just save a plot or a result, do you know exaclty which steps you needed? If you’re a really organized person, this might be possible with naming conventions. But again, R Markdown can make your life so much easier: You can save your results (e.g.&nbsp;a plot or a table) and the code that created it, in one file.</p></li>&#10;<li><p><strong>Automated reports:</strong> With R Markdown, you can automate analyses that need to be consistent and in the same format every time.</p></li>&#10;</ul>&#10;<div id="so-how-does-it-work" class="section level3">&#10;<h3>So how does it work?</h3>&#10;<p>In an R Markdown File (.Rmd) you can combine <strong>R code chunks</strong> and <strong>text</strong> (which is written in a Markdown language. More in a second!). Your R code will be excecuted and your results will be displayed. Everything is “knitted” together to a Markdown format, which can be converted into almost any format through Pandoc. The outputfile may contain your code, formated text (executed Markdown) and the R output (tables, plots, numbers…). This may sound complicated. But all you have to do is write this file, choose your output format and press the “Knit” button in R studio.</p>&#10;<p>Sounds magical? Let’s create your first R Markdown file in R Studio. I encourage you to use R Studio, because it provides a lot of awesome features, which simplify working with R Markdown.</p>&#10;<ol style="list-style-type: decimal">&#10;<li><p>Go to File –&gt; New File –&gt; R Markdown.</p></li>&#10;<li><p>Choose a Title. It does not matter and can be changed easily afterwards</p></li>&#10;<li><p>Choose your desired output (Word, Pdf…) format. Let’s choose html as a start!</p></li>&#10;<li><p>(Maybe do you need to install additional packages, R Markdown will tell you which)</p></li>&#10;</ol>&#10;<p>There is your first R Markdown file. It already contains a little code. Let’s talk about the structure of an R Markdown document.</p>&#10;</div>&#10;<div id="rmd-file-strucutre" class="section level1">&#10;<h2>.Rmd File Strucutre</h2>&#10;<p>There are 3 different kind of code.</p>&#10;<ol style="list-style-type: decimal">&#10;<li>YAML Header</li>&#10;<li>Markdown</li>&#10;<li>R Code Chunks</li>&#10;</ol>&#10;<p>If you press the “Knit” button, you see, that this runs perfectly and creates an output file. You just successfully compiled your first .Rmd file! A window with your output file should open up. Let’s look at the 3 different kinds of code in your .Rmd example file.</p>&#10;<div id="yaml-header" class="section level3">&#10;<h3>YAML Header</h3>&#10;<p>The header is on top of every .Rmd file. The code is enclosed with 3 dashes. Your Header contains meta data about your .Rmd file, such as the author, date and output format. You can add additional stuff such as gloabal variables. But don’t worry about your header too much for now. Let it just be there.</p>&#10;<pre class="r"><code>--- &#10;title: "My first R Markdown document"&#10;author: "Heike W"&#10;date: '2018-11-05'&#10;output: html_document&#10;---</code></pre>&#10;</div>&#10;<div id="markdown-text" class="section level3">&#10;<h3>Markdown (Text)</h3>&#10;<p>Markdown is a simple textformatting language. Similar to HTML or Latex, you write plain text. Punctuation characters indicate the formating. For example a <code># header1</code> indicates a header on the top level. You can make you text italic by enclosing it in 2 asterisk (<code>*italic*</code>). If you have never worked with HTML, Latex or Markdown before, this may appear strange to you: Your text appears to have no format. You cannot see how your formatting looks like until you compile the document. You have to click on the “Knit” button to compile your R Markdown. Then you will see the formating of your output file.The good thing is that you can be sure, that a header level 1 is a header level 1 and all header level 1 will look the same in the final document. Here is an overview of the most common formating:</p>&#10;<div class="figure">&#10;<img src="/post/2015-07-23-r-rmarkdown_files/RMarkdownLang.png" width="400" alt="Illustration for R Markdown for Novices: All you need to know to get started">&#10;&#10;</div>&#10;<p>Source <a href="https://www.rstudio.com/wp-content/uploads/2015/02/rmarkdown-cheatsheet.pdf">RMarkdown Cheat Sheet</a></p>&#10;</div>&#10;<div id="r-code-chunks" class="section level3">&#10;<h3>R Code Chunks</h3>&#10;<p>Just write your R code and wrap it around 3 apostrophies followed by <code>{r}</code>. This tells R Markdown that you want to write R code (You could also insert for example python <code>{python}</code>, bash <code>{bash}</code> or sql {<code>sql}</code>). Insert as many code chunks as you want. Instead of typing 3 apostrophies and the curly braces everytime, you can simply press <em>command-option-I</em> on the Mac and <em>Ctrl-Alt-I</em> on Windows to create a new code chunk. Per default, R will execute this code chunk and display the code and the results in the output file when you “knit” your .Rmd file.</p>&#10;<pre class="r"><code>```{r headCars}&#10;summary(cars)&#10;```</code></pre>&#10;<p>You have previously executed the example code by clicking on the “Knit” button in R Studio. This opens a new session and runs the entire .Rmd file. This means you’re starting with a new clean environment. You can also run single R code chunks separately. Click on the little green triange in the right corner of your code chunk and only this chunk will run. You can see, that the results are shown directly below your code.</p>&#10;<p>You can give names to your code chunks within the curly braces of your R code chunks. I named our first code chunk “headCars”. In addition, you can pass <em>options</em>. Options tell R what it should do with this code chunk. For example, for not displaying any R code in the output file pass <code>echo = FALSE</code>:</p>&#10;<pre class="r"><code>```{r headCars, echo = FALSE}&#10;summary(cars)&#10;```</code></pre>&#10;<p>Or you can tell R not to run a code chunk by setting <code>eval = FALSE</code>.</p>&#10;<pre class="r"><code>```{r headCars, eval = FALSE}&#10;summary(cars)&#10;```</code></pre>&#10;<p>You can use as many options as neccessary. The order of options does not matter, but you have to separate them with a comma.</p>&#10;<pre class="r"><code>```{r headCars, echo = TRUE, eval = FALSE}&#10;summary(cars)&#10;```</code></pre>&#10;<p>Here is a brief overview of the options, I use the most.</p>&#10;<table style="width:100%;">&#10;<colgroup>&#10;<col width="40%">&#10;<col width="36%">&#10;<col width="22%">&#10;</colgroup>&#10;<thead>&#10;<tr class="header">&#10;<th>How should the output look like?</th>&#10;<th>How to change the parameter</th>&#10;<th>Default option?</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr class="odd">&#10;<td>include code</td>&#10;<td>echo = TRUE</td>&#10;<td>yes</td>&#10;</tr>&#10;<tr class="even">&#10;<td>automatically format code</td>&#10;<td>tidy = TRUE</td>&#10;<td>no</td>&#10;</tr>&#10;<tr class="odd">&#10;<td>do not execute code</td>&#10;<td>eval = FALSE</td>&#10;<td>no</td>&#10;</tr>&#10;<tr class="even">&#10;<td>execute code, but show nothing</td>&#10;<td>include = FALSE</td>&#10;<td>no</td>&#10;</tr>&#10;<tr class="odd">&#10;<td>do not show any warnings</td>&#10;<td>warings = FALSE</td>&#10;<td>no</td>&#10;</tr>&#10;<tr class="even">&#10;<td>do not show any errors</td>&#10;<td>error = FALSE</td>&#10;<td>no</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>You can find a comprehensive overview about code chunk options on <a href="https://yihui.name/knitr/options/">Yihui Xie’s website</a></p>&#10;<p>You’re wondering whether you have to pass the same options to all your code chunks? You don’t. You can determine <strong>global options</strong> at the beginning of your .Rmd file Here in this example, where I told R Markdown to not print any code.</p>&#10;<pre class="r"><code>```{r globaloptions, include=FALSE}&#10;knitr::opts_chunk$set(echo = FALSE)&#10;```</code></pre>&#10;<p>The code chunk options overwrite the global options. So if there is one code chunk, where you want to display code, you can pass the option <code>echo = TRUE</code> to this code chunk and the code will be shown in the output document.</p>&#10;</div>&#10;</div>&#10;<div id="insert-plots-figures" class="section level1">&#10;<h2>Insert plots &amp; figures</h2>&#10;<p>You can easily insert the figures that are created with R code. Just write R code within your code chunks, that creates a plot. R Markdown will insert your plot right after your code.</p>&#10;<pre class="r"><code>```{r firstPlot}&#10;plot(cars$speed, cars$dist)&#10;```</code></pre>&#10;<p><img src="/./post/2015-07-23-r-rmarkdown_files/figure-html/firstPlot2-1.png" width="672" alt="Illustration for R Markdown for Novices: All you need to know to get started"></p>&#10;<p>As you have done previously, you can use the options of code chunks to alter your output. This works for figures as well. Most figure options start with <code>fig.</code>. For example, you can change the size (<code>fig.height, fig.width</code> (default size = 7 inches)), the alignment (<code>fig.align</code> (“left”,“right”,“center”)) and the caption (<code>fig.cap</code>) of your plot.</p>&#10;<p>Let’s create a wide plot, which with central alignment give it a caption.</p>&#10;<code>r ```{r myWidePlot, fig.height = 4, fig.align = "center", fig.cap = "my wide plot"} plot(cars$speed, cars$dist) ```</code>&#10;<div class="figure" style="text-align: center"><span id="fig:unnamed-chunk-1"></span>&#10;<img src="/./post/2015-07-23-r-rmarkdown_files/figure-html/unnamed-chunk-1-1.png" alt="my wide plot" width="672">&#10;<p class="caption">&#10;Figure 1: my wide plot&#10;</p>&#10;</div>&#10;<p>What about local images or GIFS from the internet? You can include them as well. Here is an example. The brackets are optional, if you want a title for the figure. Paste the path to the image into the braces.</p>&#10;<pre class="markdown"><code>![My plot caption](https://media.giphy.com/media/6rzRsF6ghqxSE/giphy.gif)</code></pre>&#10;<div class="figure">&#10;<img src="https://media.giphy.com/media/6rzRsF6ghqxSE/giphy.gif" alt="My plot caption">&#10;<p class="caption">My plot caption</p>&#10;</div>&#10;</div>&#10;<div id="insert-tables" class="section level1">&#10;<h2>Insert Tables</h2>&#10;<p>Per default, tables look pretty ugly. R will print the output from the console with leading <code>#</code>.</p>&#10;<pre class="r"><code>```{r carsTable}&#10;head(cars)&#10;```</code></pre>&#10;<pre><code>##   speed dist&#10;## 1     4    2&#10;## 2     4   10&#10;## 3     7    4&#10;## 4     7   22&#10;## 5     8   16&#10;## 6     9   10</code></pre>&#10;<p>You can get rid of the <code>#</code> by using the option <code>comment = FALSE</code>.</p>&#10;<pre class="r"><code>```{r carsTable, comment= FALSE}&#10;head(cars)&#10;```</code></pre>&#10;<pre><code>FALSE   speed dist&#10;FALSE 1     4    2&#10;FALSE 2     4   10&#10;FALSE 3     7    4&#10;FALSE 4     7   22&#10;FALSE 5     8   16&#10;FALSE 6     9   10</code></pre>&#10;<p>Hm, still not very pretty! The <code>kable</code> function from the <code>knitr</code> package provides nicer looking tables.</p>&#10;<pre class="r"><code>```{r carsTable, comment= FALSE}&#10;library(knitr)&#10;kable(head(cars))&#10;```</code></pre>&#10;<table>&#10;<thead>&#10;<tr class="header">&#10;<th align="left">speed</th>&#10;<th align="left">dist</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr class="odd">&#10;<td align="left">4</td>&#10;<td align="left">2</td>&#10;</tr>&#10;<tr class="even">&#10;<td align="left">4</td>&#10;<td align="left">10</td>&#10;</tr>&#10;<tr class="odd">&#10;<td align="left">7</td>&#10;<td align="left">4</td>&#10;</tr>&#10;<tr class="even">&#10;<td align="left">7</td>&#10;<td align="left">22</td>&#10;</tr>&#10;<tr class="odd">&#10;<td align="left">8</td>&#10;<td align="left">16</td>&#10;</tr>&#10;<tr class="even">&#10;<td align="left">9</td>&#10;<td align="left">10</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>In the next blogpost, I will go into more detail how you can style your tables.</p>&#10;</div></div>
