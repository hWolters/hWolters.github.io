<div class="legacy-content"><style> table { width:80% !important;} </style>&#10;<p>When I started working with pandas I noticed that there were so many ways how to subset, filter and join data with pandas. But I was lacking a systematic overview. How do the different approaches differ and when to use which?</p>&#10;<p>In this blogpost we’ll look at different ways for subsetting, filtering and combining DataFrames.</p>&#10;<h2 id="subsetting-data-selecting-subsets-of-rows-and-columns-by-labels-and-positions">Subsetting Data: Selecting subsets of rows and columns by labels and positions</h2>&#10;<p><code>.iloc</code> stands for integer-location based indexing. Which means it needs integer input and refers to the position of a row or column you want to select. You can pass integers (<code>0</code>) to select a single row, list of integers (<code>[0,4,9]</code>) and slice objects (<code>1:2,3:5</code>). Here are some examples:</p>&#10;<pre><code># rows&#10;df.iloc[0] # first row of data frame &#10;df.iloc[1] # second row of data frame &#10;df.iloc[-1] # last row of data frame &#10;df.iloc[0:10] # first 10 rows of a data frame&#10;</code></pre><pre><code># columns:&#10;df.iloc[:,0] #first column&#10;df.iloc[:,1] # second column &#10;df.iloc[:,-1] # last column of data frame &#10;df.iloc[:,0:10] # first 10 column of data frame &#10;</code></pre><p><code>.loc</code> is label based. You can use a single label (<code>'id'</code>), a list of labels (<code>['column_a', 'column_b']</code>) or a slice object (<code>row_a:row_c, column_b</code>:<code>column_d</code>). Note that if you pass an integer value to <code>.loc</code>, it will be interpreted as a label.</p>&#10;<p>If you want to select rows or columns that contain a certain string (e.g. a prefix or postfix) you can use <code>filter()</code>. You pass the string you are looking for to the parameters <code>like</code> or <a href="https://docs.python.org/3.3/howto/regex.html"><code>regex</code></a>. The axis parameter determines whether you search in the index (<code>axis = 0</code>) or in the column names (<code>axis = 1</code>). You can filter all columns that contain the string <code>foo</code> with <code>df.filter(like ='foo', axis = 1</code>.</p>&#10;<p>Here a brief overview about subsetting data</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th style="text-align:left">index</th>&#10;<th style="text-align:left">how</th>&#10;<th style="text-align:left">Example with slice object</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td style="text-align:left"><code>.iloc</code></td>&#10;<td style="text-align:left">by index (row)</td>&#10;<td style="text-align:left"><code>df.iloc[&lt;row_nr_start&gt;:&lt;row_nr_end&gt;, '&lt;col_nr_start&gt;':'&lt;col_nr_end&gt;']</code></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>.loc</code></td>&#10;<td style="text-align:left">by label</td>&#10;<td style="text-align:left"><code>df.loc[&lt;row_label_start&gt;:&lt;row_label_end&gt;, '&lt;col_name_start&gt;':'&lt;col_name_end&gt;']</code></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>filter</code></td>&#10;<td style="text-align:left">by label and regexes</td>&#10;<td style="text-align:left"><code>df.filter(like =&lt;some_string&gt;, axis = 0</code>)</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<br>&#10;<h3 id="compare-iloc-loc-and-filter">Compare <code>iloc</code>, <code>loc</code> and <code>filter</code></h3>&#10;<p>All three ways above allow you to subset your data. If your columns follow a pattern rather than a position or a name, I’ll suggest you use <code>filter</code> as it is less error prone.</p>&#10;<h2 id="filtering-data-selecting-subsets-of-rows-based-on-conditions">Filtering Data: Selecting subsets of rows based on conditions</h2>&#10;<p>There are multiple ways how to filter data with pandas. We’ll look at how boolean indexing works and how to filter with the <code>query()</code> function.</p>&#10;<p>If you are familiar with SQL, <code>query</code> might be the most intuitive way for you. The idea is to use a boolean mask to index the <code>DataFrame</code>. This boolean mask is a vector of true and false values for each row in the <code>DataFrame</code>.</p>&#10;<pre><code>new_df = df[&lt;some boolean mask&gt;]&#10;</code></pre><p>Only the rows with true values will be returned. id="combining-data-joins">Combining Data: joins</h2>&#10;<p>Pandas has three functions for joining tables <a href="https://pandas.pydata.org/pandas-docs/stable/reference/api/pandas.DataFrame.join.html"><code>join</code></a>,<a href="https://pandas.pydata.org/pandas-docs/stable/reference/api/pandas.DataFrame.merge.html"><code>merge</code></a> and <a href="https://pandas.pydata.org/pandas-docs/stable/reference/api/pandas.concat.html"><code>concat</code></a>. <code>join</code> and <code>merge</code> are similar to joining tables in SQL with a (left, right etc….) join. <code>join()</code> will attempt to do a left join on indices keeping all columns while <code>merge</code> will perform an inner join on the common columns.  Both can combine two tables. <code>concat</code> is the most flexible option, which allows you to join rows <strong>and</strong> columns of multiple DataFrames. Your datasets are just stitched together along an axis — either the row axis or column axis- ignoring the index. So you can use <code>concat</code> for joining and for the union of DataFrames.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>function</th>&#10;<th>combines</th>&#10;<th>default</th>&#10;<th>how</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td><code>join</code></td>&#10;<td>2 dataframes by indices</td>&#10;<td>left</td>&#10;<td>left, right, inner, outer</td>&#10;</tr>&#10;<tr>&#10;<td><code>merge</code></td>&#10;<td>2 dataframes by columns or index</td>&#10;<td>inner</td>&#10;<td>…</td>&#10;</tr>&#10;<tr>&#10;<td><code>concat</code></td>&#10;<td>n dataframes by columns, index, or rows</td>&#10;<td>append rows</td>&#10;<td>…</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<br>&#10;<h3 id="compare-join-vs-concat-vs-merge">Compare <code>join</code> vs. <code>concat</code> vs. <code>merge</code></h3>&#10;<p>Let’s create some datasets, what would happen if we combine those datasets with their default parameters. Note that the indices differ!</p>&#10;<pre><code>A = pd.DataFrame({'ID': [1, 2, 3, 4],&#10;                    'Name': ['Tom', 'Lisa', 'Sara', 'John'],&#10;                    'Age': ['10', '11', '12', '11']},&#10;                   index=[0, 1, 2, 3])&#10;&#10;&#10;B = pd.DataFrame({'ID': [1,3,4,6],&#10;                  'Gender': ['m', 'f', 'm', 'm']},&#10;                  index=[0, 1, 5, 6])&#10;&#10;</code></pre><h3 id="join"><code>join</code></h3>&#10;<pre><code>A.join(B)&#10;</code></pre><p>will throw an error <code>columns overlap but no suffix specified: Index(['ID'], dtype='object')</code>. Since we have the <code>ID</code> Column in both datasets, we have to specify a suffix.</p>&#10;<pre><code>A.join(B, lsuffix = "a")&#10;&#10;&#10;   	IDa	 Name	Age	ID 		Gender&#10;0	1	 Tom	10	1.0 	m&#10;1	2	 Lisa	11	3.0 	f&#10;2	3	 Sara	12	NaN		NaN&#10;3	4	 John	11	NaN		NaN&#10;</code></pre><p>Note that <code>join</code> does a left join on the index and completely ignores the <code>ID</code> column.</p>&#10;<h3 id="merge"><code>merge</code></h3>&#10;<p>Merge per default performs an inner join on the common columns. It does not care about the index.</p>&#10;<pre><code>A.merge(B)&#10;&#10;&#10;   	ID 	Name	Age	Gender&#10;0	1	Tom		10	m&#10;1	3	Sara	12	f&#10;2	4	John	11	m&#10;</code></pre><p>Using the how parameter, you can determine the type of join</p>&#10;<pre><code>	ID		Name	Age		Gender&#10;0	1		Tom	    10		m&#10;1	2		Lisa	11		NaN&#10;2	3		Sara	12		f&#10;3	4		John	11		m&#10;</code></pre><h3 id="concat"><code>concat</code></h3>&#10;<p>Per default, <code>concat</code> will create a union of the datasets, ignoring the index. It just unions both <code>DataFrames</code> and creates <code>NaN</code>s for columns that are missing in one of the datasets.</p>&#10;<pre><code>pd.concat([A, B])&#10;&#10;	ID	Name		Age		Gender&#10;0	1	Tom	    	10		NaN&#10;1	2	Lisa		11		NaN&#10;2	3	Sara		12		NaN&#10;3	4	John		11		NaN&#10;0	1	NaN	    	NaN		m&#10;1	3	NaN  		NaN		f&#10;5	4	NaN	    	NaN		m&#10;6	6	NaN 		NaN		m&#10;</code></pre><p>I typically use <code>merge</code> for joining <code>DataFrames</code> in a SQL-like way and <code>concat</code> for stitching <code>DataFrames</code> together or SQL-like <code>union</code>s.</p>&#10;<h2 id="conclusion">Conclusion</h2>&#10;<p>We have seen several ways how to subset, filter and join DataFrames. Some of the functions will get you to the same results, while others have small differences you need to be aware of.</p></div>
