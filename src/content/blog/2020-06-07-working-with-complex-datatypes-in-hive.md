<div class="legacy-content"><style> table { width:80% !important;} </style>&#10;<p>The basic idea of complex datatypes is to store multiple values in a single column. So if you are working with a Hive database and you query a column, but then you notice “This value I need is trapped in a column among other values…” you just came across a complex a.k.a. nested datatype.</p>&#10;<p>There are three types: <code>arrays</code>, <code>maps</code> and <code>structs</code>. First, you have to understand, which types are present. Depending on the datatype, there are different ways how you can access the values.</p>&#10;<ul>&#10;<li>&#10;<p><code>array</code>(<code>ARRAY&lt;data_type&gt;</code>): It is an ordered collection of elements. The elements in the array must be of the same type. So you have for example an array of strings, integers etc.</p>&#10;</li>&#10;<li>&#10;<p><code>map</code> (<code>MAP&lt;primitive_type, data_type&gt;</code>): It is an unordered collection of key-value pairs. Keys must be of primitive types. Values can be of any type. A map can have an arbitrary number of elements.</p>&#10;</li>&#10;<li>&#10;<p><code>struct</code> (<code>STRUCT&lt;col_name : data_type&gt;</code>): It is a collection of elements of different types. The concept of a struct is very similar to a row in a table: structs have a fixed number of fields (“columns”). Every field has a predefined type.</p>&#10;</li>&#10;</ul>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th style="text-align:left">type</th>&#10;<th style="text-align:left">what is it?</th>&#10;<th style="text-align:left">how do they look like</th>&#10;<th style="text-align:left">example</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td style="text-align:left"><code>Array</code></td>&#10;<td style="text-align:left">ordered collection  of elements of the <em>same</em> type</td>&#10;<td style="text-align:left">a comma-separated list surrounded with […]</td>&#10;<td style="text-align:left"><code>["a","b"]</code></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>Map</code></td>&#10;<td style="text-align:left">unordered collection of key-value paris</td>&#10;<td style="text-align:left">comma-separated list of key:value pairs, surrounded with {…}</td>&#10;<td style="text-align:left"><code>{"a":1, "b":2}</code></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>Struct</code></td>&#10;<td style="text-align:left">collection of elements of different types</td>&#10;<td style="text-align:left">written using the JSON map format</td>&#10;<td style="text-align:left"><code>{"a": 1, "b":"c"}</code></td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>Use <code>DESCRIBE &lt;table&gt;</code> to get a list of the datatypes of the columns in your table. You may notice, that one complex data type is nested into another, e.g. you may see an array of structs. But don’t worry, if you understand each type separately, you can untangle the nested structures as well. So let’s have a look at each of those types and how we can work with them.</p>&#10;<h2 id="arrays">Arrays</h2>&#10;<p>So arrays are an ordered collection of elements of the same type. You could compare them to lists of the same type in Python or vectors in R. Since they are ordered, you can use a numeric index to access the elements. The index starts at 0.</p>&#10;<p>For example, the table <code>students</code> has a column <code>extra_curriculum</code>, which is an array of strings.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th>extra_curriculum</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td>[‘orchestra’]</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>[‘orchestra’, ‘art’]</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>You can get the first value in the array, by indexing it with <code>[0]</code>.</p>&#10;<pre><code class="language-{sql}" data-lang="{sql}">SELECT &#10;	first_name, &#10;	extra_curriculum[0] AS first_extra_curriculum &#10;FROM &#10;	students&#10;;&#10;</code></pre><p>This returns:</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th>first_extra_curriculum</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td>orchestra</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>orchestra</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>Let’s now say you want to have one row for each element in your array (The equivalent of Rs <code>gather</code>-function). Hive has build-in table generating functions (UDTF) for that. These UDTFs transform a single input row into multiple output rows. The basic syntax is</p>&#10;<p><code>LATERAL VIEW explode(expression) tableAlias AS columnAlias </code></p>&#10;<p>This returns 0 to many rows: one row for each element from the input array. So running the following query on our students table</p>&#10;<pre><code class="language-{sql}" data-lang="{sql}">SELECT &#10;	first_name, &#10;	extra_curriculum &#10;FROM &#10;	students&#10;	LATERAL VIEW EXPLODE (extra_curriculum) ec AS extra_curriculum&#10;;&#10;&#10;</code></pre><p>will result in a table, with each entry in <code>extra_curriculum</code> as a row.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th>extra_curriculum</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td>orchestra</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>orchestra</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>art</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<h2 id="maps">Maps</h2>&#10;<p>Maps are used for key-value pairs. They are very similar to dictionaries in Python or a named vector in R. You can access the key-value pairs  with the name of the key in brackets.</p>&#10;<p>In this example, you have the table <code>students</code> with column <code>grade</code>, <code>MAP&lt;string, string&gt;</code>, which maps different subjects to their letter grade.</p>&#10; <br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th style="text-align:left">grade</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td style="text-align:left">{‘math’: ‘B’, ‘english’: ‘B’}</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td style="text-align:left">{‘math’: ‘A’, ‘english’: ‘B’, ‘biology’: ‘C’}</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<p>You can extract all the math grades by using the key <code>math</code> in brackets.</p>&#10;<pre><code class="language-{sql}" data-lang="{sql}">SELECT &#10;	first_name, &#10;	grade["math"] AS math_grade &#10;FROM &#10;	students&#10;;&#10;</code></pre><table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th style="text-align:left">math_grade</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td style="text-align:left">B</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td style="text-align:left">A</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<h2 id="structs">Structs</h2>&#10;<p>Structs are written in JSON format. You can access the values using the dot notation for the field to extract the value.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th>teacher</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td>{‘math’: ‘Mrs Johnson’, ‘english’: ‘Mr Miller’, ‘nr_teachers’: 2}</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>{‘math’: ‘Mrs Johnson’, ‘english’: ‘Mrs Thomson’, ‘biology’: ‘Mr Chu’, ‘nr_teachers’: 3}</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<pre><code class="language-{sql}" data-lang="{sql}">SELECT &#10;	first_name, &#10;	teacher.math AS math_teacher &#10;FROM &#10;	student&#10;;&#10;</code></pre><table>&#10;<thead>&#10;<tr>&#10;<th>first_name</th>&#10;<th>math_teacher</th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td>Tom</td>&#10;<td>Mrs Johnson</td>&#10;</tr>&#10;<tr>&#10;<td>Ann</td>&#10;<td>Mrs Johnson</td>&#10;</tr>&#10;</tbody>&#10;</table>&#10;<h2 id="summary">Summary</h2>&#10;<p>To sum it up, depending on the complex data type, you can query elements differently. This last table gives an overview on how.</p>&#10;<br>&#10;<table>&#10;<thead>&#10;<tr>&#10;<th style="text-align:left">Type</th>&#10;<th style="text-align:left">How to extract elements</th>&#10;<th style="text-align:left">Example</th>&#10;<th></th>&#10;</tr>&#10;</thead>&#10;<tbody>&#10;<tr>&#10;<td style="text-align:left"><code>Array</code></td>&#10;<td style="text-align:left">by index</td>&#10;<td style="text-align:left"><code>my_array[0]</code></td>&#10;<td></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>Map </code></td>&#10;<td style="text-align:left">by key</td>&#10;<td style="text-align:left"><code>my_map['key']</code></td>&#10;<td></td>&#10;</tr>&#10;<tr>&#10;<td style="text-align:left"><code>Struct </code></td>&#10;<td style="text-align:left">by dot notation</td>&#10;<td style="text-align:left"><code>my_struct.name</code></td>&#10;<td></td>&#10;</tr>&#10;</tbody>&#10;</table></div>
